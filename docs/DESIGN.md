# archscope — design notes

Date: 2026-06-11. Decisions made with Haozhe via question round; survey by web agent
(claude), lingbot-va analysis verified by hand against the code.

## Product decisions (user-confirmed)

| question | decision | rationale |
|---|---|---|
| form factor | **static generator** (Python → SVG/PDF per hierarchy level), not a web viewer | zero frontend deps, CI-able, outputs are reviewable/committable artifacts; interactivity deferred |
| model → spec | **agent reads code, writes the figure** (no tracing required) | works without an environment that can instantiate the model; semantic grouping quality is the whole game; tracer later as a shape cross-check |
| training-method figures | **hand-written first, extract reusable primitives later** | per-method quality ceiling matters more than generality early; TokenRow/MaskGrid/Formula were the first extractions |
| repo name | archscope | — |

## Survey: why the niche is empty (2026-06)

Closest per requirement — R1 paper-grade hierarchical block diagrams: *Net2Vis*
(code→abstracted SVG with a visual grammar mined from CVPR figures; Keras/CNN-only,
dormant), *Model Explorer* (best hierarchy interaction, WebGL, debugger aesthetics,
no vector export), *PlotNeuralNet* (best raw output, fully manual, frozen 2018).
R2 shape-annotated flow: *Netron* (active, universal, dagre-based; op-soup at scale),
*torchview / torchlens* (PyTorch tracing; graphviz looks). R3 training-method
illustration: **nothing** — poloclub explainers are hardcoded one-offs; SciFig /
PaperBanana (2026) are text→raster VLM agents, not code-grounded. Agent-reads-repo
exists (GitDiagram, DeepWiki, CodeBoarding) but only for software architecture in
Mermaid. Five-way intersection: empty.

Key takeaways we stole: Net2Vis's "visual grammar from conference figures" framing;
HiddenLayer's graph-folding rules → our known-concept collapse; Model Explorer's
per-container lazy layout → our per-figure hand layout; "export = serialize the SVG"
as the publication-quality strategy.

## Architecture

```
src/archscope/
  style.py       palette (kind→pastel family), modality colors, edge styles, type scale
  geom.py        Box + side/fraction anchors
  svg.py         SvgDoc: 4 paint layers (frames<edges<nodes<labels), markers, hatch
                 patterns, bbox tracking, resvg PNG export
  text.py        DejaVu-metric measurement (slightly wide ⇒ boxes never underflow);
                 mathtext→outlined SVG paths (Computer Modern), so exports are
                 self-contained
  node.py        Block(label/sub/src/badge/modality-stripe), OpDot(+,×,⊙ drawn as
                 vector, not font glyphs), IOLabel(stadium tensor), KnownChip(✓),
                 TextLabel(wrapping), Formula, Spacer
  containers.py  VStack/HStack(center|start|end), Free(absolute), GroupFrame(titled),
                 RepeatFrame(×N tag)
  edges.py       polyline routing (straight/vh/hv/zv/zh/via), rounded corners,
                 per-color arrow markers, halo'd mono shape labels
  diagram.py     anchor registry ("id.b@0.25"), facing-side auto-resolution, chain(),
                 note(), title block, save
figures/<model>/ figure scripts = the spec (Python). common.py per model.
out/<model>/     svg + png + README.md gallery — committed artifacts.
```

Layout philosophy: **the agent is the layout engine.** Paper figures are
hand-positioned; auto-layout (elk/dagre/graphviz) is what makes Netron look like
Netron. Measurement-based stacks + explicit coordinates + anchor-based edges hit the
quality bar at the cost of per-figure effort — the accepted trade.

## Authoring methodology (the part that generalizes beyond this repo)

1. **Verify-then-draw.** Subagents map the repo/paper; the author re-reads every
   load-bearing line before it becomes a node. (This caught the Explore agent's wrong
   "all weights shared incl. modality embedders" summary and the paper/code MoT split.)
2. **Decomposition ladder.** L0 system → L1 model dataflow → L2 block → L3 sublayer;
   each figure points to the next (`→ Fig N` in src lines). Complex models = several
   figures, not one mural.
3. **Known-concept collapse.** Standard components the reader knows render as
   KnownChip / a one-line sub instead of expansion (here: SDPA inside WanAttention;
   the reader was assumed to know MHA but not MoT).
4. **Computed truth.** Mask grids / schedules are generated by re-implementing the
   code's rules inside the figure script (fig05), never hand-transcribed.
5. **Conflicts are content.** Paper-vs-code differences get their own figure (fig07)
   with the evidence (fossil module names) on the canvas.

## Pitfalls log

- resvg font fallback: Helvetica Neue → Arial Unicode MS → **.LastResort** (tofu).
  Banned: ₜ-style letter subscripts, combining accents (ẑ z̃ v̂), ∥. Allowed: ₀-₉,
  Greek, arrows, ≤ ∈ ≈ ①②. Real math → `Formula` (outlined paths, immune).
- zsh: `echo ===` is parsed as a glob — quote it.
- `Diagram._pt` splits anchor specs on the FIRST dot ("id.b@0.25" — fractions contain
  dots).
- cairosvg needs system libcairo on macOS; resvg (brew) is the dependable PNG path.
- Edge labels default to the longest segment's midpoint — for short multi-segment
  conditioning edges, prefer a bus (one trunk + drawn rail + labeled branches, fig03)
  over four labeled edges.
- Style rules from review (2026-06-11): arrows perpendicular to box sides with entry/
  exit stubs (default `_perp_route`); repeated blocks as overlapping card stacks with
  dashed ghosts (`RepeatStack`), edges attach to the cluster envelope so they never
  cross the ghost outlines.

## Legibility playbook (from a conference-reviewer audit, 2026-06-13)

A harsh NeurIPS-reviewer review of the figure set found the *example* figures
(qwen3, the flowers, dit) legible but the *architecture* figures (the LingBot set)
confusing: the reader "couldn't tell what the whole is, what each block's input is,
or how parts relate." The recurring failures and the rules that fix them:

1. **Show the whole, don't tell it.** Every figure needs a real input→output spine:
   one labeled INPUT pill and one labeled OUTPUT pill at the two ends (shape on both),
   plus a one-line "this is X: A→B" banner. Don't bury the system in the subtitle or
   split it into parallel rails with no single path.
2. **Canvas ≤ ~2:1.** Re-flow wide pipelines into stacked rows / a vertical spine.
   A 3.8:1 strip is unreadable at page width.
3. **Type every edge, on the arrow; run a collision pass.** No label/legend/badge may
   overlap a box; put each tensor shape *on its arrow* with a leader. (The worst bug
   found was a legend smeared over an output pill — keep legends out of the content.)
4. **One block = name + (in-shape → out-shape) + ≤4-word role.** Move sentences,
   `path:line`, and caveats to the caption or a side-note. No paragraphs inside boxes.
5. **Show order with numbered, directional arrows; draw the two loop kinds differently**
   (an "×N layers" stack bracket vs an "×T sampler / next-chunk" back-edge must not look
   alike). Sequence ①→② as edges, not prose.
6. **Collapse conditioning into one bus** that taps the block at *named ports*
   (`→ AdaLN`, `→ cross-attn K/V`). Kill floating t/c/RoPE/mask boxes with crossing
   dashed lines.
7. **Every figure self-contained.** Inline a 3-box stub of any referenced block instead
   of "→ Fig N"; cross-refs supplement, never substitute the centerpiece.
8. **For comparisons, align on one shared spine and diff only the delta** (don't draw
   two full architectures side by side); let a table carry the specifics.
9. **Concrete > abstract for pedagogy.** Draw real tokens (image thumbnails, value
   chips) rather than `z1 a1 z2` boxes when illustrating a sequence — the flower
   figures are legible because of this.

Applied to: dit (legend fix), fig10/fig02/fig09/fig01 (full rebuilds to the spine +
bookend-pills + conditioning-bus discipline).

## Low-friction API (studied diagrams.py / mermaid / graphviz / d2, 2026-06-13)

The other lesson: those tools never make you write coordinates — you declare
structure (nodes + edges) and a layout engine positions them. archscope keeps the
"agent is the layout engine" thesis (hand-placed for paper-grade control) but adopts
the lightweight wrappers that cut the render→eyeball-overlaps→nudge loop:

- `d.flow([...], dir="up")` — one call = stack + auto-id-from-label + auto-connect
  consecutive nodes (the diagrams.py `>>` idea). Children in dataflow order.
- relative placement `d.place(el, right_of="enc", gap=80, ref_align="cy")` — kills the
  inter-column magic numbers that drift into overlap on edits (graphviz `rank=same` /
  d2 `near`, as a ~15-line `place()` upgrade).
- `d.check() -> [(a,b,frac)]` over all id'd boxes + `save(on_overlap="warn"|"raise")` —
  the overlap linter is a machine assertion, not a human eyeballing the PNG. It reports
  three defect classes: box overlaps (`check`), arrowhead pile-ups (`check_arrows`, two
  heads landing within ~7px), and edges passing through a box (`check_edges_through_boxes`).
- obstacle-aware auto-router: a plain `d.edge("a", "b.l@0.25")` first tries the straight
  perpendicular path, and **if that would pierce another box it detours automatically** —
  sweeping the bend, then re-picking the exit side of any bare-id endpoint (a pinned
  `id.side@frac` port keeps its landing point). A default-clean route is returned
  verbatim, so the common case never shifts. This is the "generate correctly the first
  time" lever: the same geometry test (`_poly_pierces`) the router routes *away from* is
  the one the linter flags, so what renders clean is clean by the identical rule. The
  author declares intent (`a -> b`) and does not hand-route around obstacles.
  Locked in by `tests/test_routing.py`. Limit: only boxes placed *before* the edge are
  seen as obstacles; the save()-time linter is the backstop for the rest.
- `d.auto_legend()` scans the kinds/modalities/edge-styles actually used; `Swatches`
  keys edge styles (short arrows) and op glyphs so every special symbol is in the key.

Not adopted (deliberately): a graphviz hard dependency. If auto-layout is ever wanted,
the minimal path is a pure-Python longest-path layered placer (or `networkx.
multipartite_layout`), or shelling to `dot -Tplain` for positions only — off by default.
